# NAICS Coder
https://bryncoll.github.io/NAICSCoder/#/level/2

A static React web app for browsing the 2022 NAICS industry hierarchy and, for each 6-digit national industry, viewing the top NAPCS products by Economic Census revenue.

Built to replace an AppSheets prototype using the same underlying data.

## Web app

### First-time setup

```bash
# From the repo root — convert xlsx to JSON (requires Python 3, no extra packages)
python3 scripts/parse_data.py

# Install JS dependencies
cd webapp && npm install
```

### Development

```bash
cd webapp && npm run dev
# Open http://localhost:5173
```

### Production build

```bash
cd webapp && npm run build
# Output in webapp/dist/ — open dist/index.html directly or deploy to any static host
```

The build uses `base: './'` so it works when opened as a local file without a server.

### Features

- **Five hierarchy levels**: Sectors → Subsectors → Industry Groups → Industries → National Industries (6-digit)
- **Search**: Filter any level by code or label
- **Detail card**: Full description, cross-references, revenue, and parent/child navigation
- **NAPCS product rankings**: Top-5 products by 2022 Economic Census revenue shown on every 6-digit NAICS card (sourced from `NAPCS_ranks.xlsx`)

## Data sources

### NAICS_Coder_V3.xlsx — Sheet1

The NAICS 2022 hierarchy. 2,125 rows across five code levels:

| Level | Name | Count |
|-------|------|-------|
| 2 | Sectors | 20 |
| 3 | Subsectors | 96 |
| 4 | Industry Groups | 308 |
| 5 | Industries | 689 |
| 6 | National Industries | 1,012 |

Columns: Code Level, Code, Code Label, Description, Cross-References, Parent Code, Revenue ($).
Revenue is sourced from the 2022 Economic Census and pre-formatted (e.g. `725.1B`).

### NAPCS_ranks.xlsx — industries sheet

One row per 6-digit NAICS code. For each industry, up to 5 top NAPCS 2022 products are listed with their product code, description, revenue ($1,000), and percentage of industry total revenue.

Key columns: NAICS, NAPCS Join Flag (explains why some codes have no data), Product1–5, Label1–5, Rev1–5 (in $1,000), Pct1–5.

The `main` sheet in this file contains the raw API output plus two manually computed columns:
- **Column J — Rank**: `RANK.EQ` over NAPCSDOL within the same industry, excluding totals and suppressed values
- **Column K — Ties (≤5)**: count of rank ties within the top-5 positions

## Census API refresh script

`refreshEcnNapcsInd.js` is a Google Apps Script that fetches data from the U.S. Census Bureau 2022 Economic Census NAPCS API and writes it to a Google Sheet.

```
Endpoint: https://api.census.gov/data/2022/ecnnapcsind
Fields:   NAPCS2022, NAPCS2022_LABEL, NAPCSDOL, NAPCSDOL_F, LINEALL_PCT, LINEALL_PCT_F
Filter:   NAICS2022=pseudo(N0600.00)  (all Mining + Utilities industries)
          TAXSTAT=00, TYPOP=00
```

To reuse the script for a different industry group, update the `NAICS2022` filter parameter and the target Sheet ID / sheet name at the top of the function. Deploy it in [Google Apps Script](https://script.google.com) and run or schedule it as needed.

### Data quality flags

The API returns flag columns (`NAPCSDOL_F`, `LINEALL_PCT_F`) that describe data quality:

| Flag | Meaning |
|------|---------|
| `s` | Withheld to avoid disclosing data for individual companies |
| `A` | Relative standard error ≥ 50% — use with caution |
| `D` | Withheld to avoid disclosing data for individual companies (disclosure avoidance) |
| `B` | Base figure withheld — percentage not computed |
| `Z` | Value rounds to zero |

B/D pairs occur when a broad NAPCS category (`B`) and a specific sub-category (`D`) report the same revenue figure; the `DL Tie Flag` column in the `main` sheet marks these pairs.
